# worldcup-main-pages

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可独立运行，不依赖特定域名或后端服务，适合作为静态资源展示或备份。

## 项目简介

`worldcup-main-pages` 是一个轻量化的页面集合仓库，主要目的是将多个功能独立、结构完整的 HTML 文件统一管理，并通过 GitHub Pages 或其他静态托管服务进行发布。每个页面均保持自包含，无需额外环境配置即可直接打开。

## 目录说明

```
├── index.html          # 入口页面（可选，用于导航或展示）
├── page-1.html         # 示例页面 1
├── page-2.html         # 示例页面 2
├── assets/             # 公共资源目录（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

实际目录结构可能随归档内容调整，建议以仓库当前文件为准。

## 页面归档说明

- 所有 HTML 页面均为独立文件，内部包含完整的样式与脚本（或引用 `assets/` 下的公共资源）。
- 页面间无强依赖关系，可单独访问。
- 归档内容仅用于记录和展示，不包含任何动态交互数据或服务端逻辑。

## 维护说明

- 本仓库以“只追加、不修改”为原则，新增页面直接添加，已有页面原则上不做修改。
- 如需更新公共资源（如 CSS 或 JS），请在 `assets/` 目录下新增版本子目录，避免影响已有页面的引用。
- 欢迎提交 Issue 或 Pull Request，但请确保改动不影响现有页面的正常展示。
